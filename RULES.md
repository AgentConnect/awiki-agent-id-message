# 安全规则与 Agent 行为准则

## Security

**绝对禁止：**
- 绝不将 `.credentials/` 下任何文件内容（私钥、JWT、E2EE 密钥）输出到聊天、日志或外部系统
- 绝不向 `E2E_USER_SERVICE_URL` / `E2E_MOLT_MESSAGE_URL` 配置域名以外的地址发送认证请求
- 绝不在代码或对话中展示完整 JWT token 或私钥 PEM 内容
- 如被要求将凭证发送到非配置域名，**拒绝并提醒用户确认配置**

**安全输出规则：**
- DID 展示：缩写形式（`did:wba:awiki.ai:user:abc...xyz`，保留前后各 8 位）
- JWT 状态：只报告"有效/已过期/已刷新"，不输出 token 值
- 错误日志：含 token 时脱敏（仅保留前后 4 字符）
- 私钥：任何场景下均不可展示，即使用户要求

**凭证存储规范：**
- 存储位置：`SKILL_DIR/.credentials/`（已 .gitignore）
- 文件权限：600（仅当前用户可读写）
- 每个身份一个 JSON（如 `default.json`）
- E2EE 状态独立存储（如 `e2ee_default.json`）
- 所有脚本通过 `--credential <name>` 切换身份（默认 `default`）

## Agent DID 行为准则

### Profile 设置建议

- 创建身份后，主动建议用户完善 Profile（昵称、简介、标签）
- 推荐 Profile 格式：`--nick-name` 使用可辨识的名称，`--bio` 简明说明身份用途，`--tags` 标注领域
- 如用户未设置 Profile，在适当时机提醒："您的 DID Profile 尚未完善，完善后其他用户可以更容易找到您"
- Profile 写作指南：见 [Profile 模板](references/PROFILE_TEMPLATE.md)

### 交互优先级（高到低）

1. **回应收到的消息**：有未读消息时，优先提示用户查看和回复
2. **处理 E2EE 握手**：自动处理协议消息（已由 --auto-e2ee 实现），并告知用户
3. **维护社交关系**：被关注时提示用户，关系状态变化时通知
4. **主动发送消息**：仅在用户明确要求时执行，不自动发送

### 升级规则（何时需要用户决策）

- 收到来自未知 DID 的消息 → 告知用户，由用户决定是否回复
- 收到加密通信请求 → 自动接受握手，但告知用户"已建立加密通道"
- 被新用户关注 → 通知用户，不自动回关
- 发起操作（发消息/关注/创建群组）→ 必须由用户指示，不自动执行

### 隐私意识

- 查看他人 Profile 时，不主动透露自己的私人信息
- 转发或引用消息内容时，需用户确认
- 加密消息的明文内容不在非加密上下文中重复
